- Sie können den Zweck und die Funktionsweise der wichtigsten historischen Netzwerkmittel (Druckerkabel, Akustikkoppler, Modem) beschreiben.
- Sie können die Bus-, Stern-, Ring- und Mesh-Topologie (voll- und teilvermascht) skizzieren und miteinander vergleichen.
- Sie können erklären, wie ein typisches Heim- oder Firmennetz aufgebaut ist.
- Sie können den Sinn eines Powerline-Adapters erklären.
- Sie können beschreiben, wie man heute und früher ins Internet gelangt(e).
- Sie können über die Entwicklung, die Relevanz und die Herausforderungen des globalen Seekabelnetzwerks Auskunft geben.
- Sie können über die Entstehung des Internets und des World Wide Web (WWW) Auskunft geben und deren Anfänge mit dem heutigen Stand der Dinge vergleichen.
- Sie können die Begriffe _Internet_ und _World Wide Web_ voneinander unterscheiden und verschiedene Internetdienste nennen.
- Sie können anhand der Begriffe _Client_ und _Server_ erklären, weshalb Daten im Internet über die gesamte Welt hinweg transportiert werden müssen.
- Sie können erklären, weshalb wir das Internet als das _Netz der Netze_ bezeichnen.
- Sie können die vier Schichten des Schichtenmodells aufzählen.
- Sie können die Hauptaufgaben jeder Schicht des Schichtenmodells nennen und können die Schichten mit einer Analogie verbinden.
- Sie können die wichtigsten Vorteile eines Schichtenmodells (Aufbau auf Bestehendem, austauschbare Elemente, Reduktion der Komplexität) nennen und erläutern.
- Sie können den Aufbau und die Aufgaben der zwei Arten von Adressen in Netzwerken (MAC und IP) vergleichen.
- Sie können den Zweck der Subnetzmaske in einem Netzwerk erklären.
- Die können anhand einer Subnetzmaske bestimmen, ob sich zwei gegebene IP-Adressen im gleichen Subnetz befinden.
- Sie können Gründe dafür nennen, weshalb wir ein Netzwerk (z.B. in einer Firma) in mehrere Subnetze unterteilen.
- Sie können die Ziele und Funktionsweisen von Routing in Netzwerken und Routing im Alltag miteinander vergleichen.
- Sie können den Zweck des `ping`-Befehls erklären und dessen Ausgabe interpretieren.
- Sie können im Output des `tracert`-Befehls die IP-Adresse des Standardgateways des ausführenden Geräts identifizieren.
- Sie können für eine gegebene URL die Domain und die TLD identifizieren.
- Sie können die Notwendigkeit des DNS (Domain Name System) erklären.
- Sie können erklären, weshalb DNS dezentral aufgebaut ist.
- Sie können den Ablauf einer DNS-Abfrage für eine gegebene Domain (inkl. Subdomains) beschreiben.
