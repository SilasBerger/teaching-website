# Kollision mit dem Fensterrand
Damit der Ball nicht gleich aus dem Spielfenster herausfliegt, wollen wir in diesem Schritt daf√ºr sorgen, dass er mit den Fensterr√§ndern kollidiert und zur√ºckprallt.

Dazu m√ºssen wir als Erstes pr√ºfen k√∂nnen, ob der Ball einen Fensterrand ber√ºhrt. Diese Funktionalit√§t setzen wir auf den Zeilen `44`-`45` f√ºr den oberen Rand und `47`-`48` f√ºr die beiden Seitenr√§nder um ‚Äî und zwar als Methoden in der `Ball`-Klasse. Der Ausdruck `self.rect.top <= 0` auf Zeile `45` in der Methode `hits_top_of_window()` pr√ºft genau das. Dieser Wert, `True` oder `False`, wird direkt _zur√ºckgegeben_.

:::insight[R√ºckgabewerte]
Was heisst, der Wert wird _zur√ºckgegeben_? Eine Funktion (oder Methode; der Einfachheit halber sprechen wir hier nun lediglich von Funktionen) kann zwei Arten von Aufgaben erf√ºllen: Sie kann etwas bewirken, oder sie kann einen Wert produzieren.

Wenn eine Funktion einen Wert produziert, dann sagen wir, dass sie einen Wert _zur√ºckgibt. Solche Funktionen haben Sie in diesem Tutorial bereits ganz oft gesehen; n√§mlich immer dann, wenn ein Funktionsaufruf rechts von einem `=`-Zeichen steht (wie z.B. auf den Zeilen `11`, `15` oder `18`).

Damit eine Funktion einen Wert zur√ºckgibt, brauchen wir das Schl√ºsselwort `return`. Der Wert, der rechts von diesem `return` steht, ist das, was diese Funktion zur√ºckgibt.

Ausf√ºhrlichere Informationen zu diesen sogenannten _R√ºckgabewerten_ finden Sie in [diesem Artikel](../../../03-Ressourcen/03-Advanced-Python/04-Return.mdx).
:::

Damit k√∂nnen wir also bereits pr√ºfen, ob der Ball die Oberkante des Spielfensters ber√ºhrt. Dasselbe machen auf den Zeilen `47`-`48` anschliessend f√ºr die Seitenr√§nder. Hier interessiert uns nicht das `self.rect.top`, sondern das `self.rect.left` und das `self.rect.right`. F√ºr ersteres pr√ºfen wir, auf Zeile `48`, ob es den linken Spielfensterrand ber√ºhrt (`self.rect.left <= 0`). F√ºr zweiteres pr√ºfen wir, ob es √ºber die Breite des Spielfensters hinausgeht und somit den rechten Rand ber√ºhrt (`self.rect.right >= window_width`). Beide Ausdr√ºcke ergeben wieder einen `True` oder `False` Wert, und wir verkn√ºpfen diese Werte mit einem `or`. Das Ergebnis, welches wir hier zur√ºckgeben, ist dann `True`, wenn `self.rect.left <= 0` oder `self.rect.right >= window_width` `True` ist.

Nun k√∂nnen wir pr√ºfen, ob der Ball √ºber den oberen, linken oder rechten Bildschirmrand hinausgeht. Diese √úberpr√ºfung rufen wir weiter unten auf Zeile `63`, respektive `66` auf und benutzen deren _R√ºckgabewert_ dort als `if`-Bedingung. Wenn der Ball den oberen Rand ber√ºhrt, dann wollen wir seine Richtung in der y-Achse umdrehen (`Zeile 64`) ‚Äî sprich, er soll nicht mehr nach oben, sondern nach unten fliegen. Wenn er einen der Seitenr√§nder ber√ºhrt, dann wollen wir seine Richtung in der x-Achse umdrehen: wenn er sich aktuell nach links bewegt, und so einen Seitenrand ber√ºhrt, dann soll er nach rechts wechseln, und umgekehrt (`67`).

Dazu fehlen uns nochmal zwei Methoden in der `Ball`-Klasse, welche diese Richtungs√§nderung bewerkstelligen. Wir implementieren sie auf den Zeilen `34`-`37`, respektive `39`-`42`. F√ºr `flip_x_direction()`, die Richtungs√§nderung in der x-Achse, berechnen wir dazu zuerst die neue x-Richtung, indem wir die aktuelle x-Richtung mal `-1` rechnen (Zeile `35`). Wenn der Ball also aktuell nach links geht (negative x-Richtung), dann soll er neu genauso schnell nach rechts gehen (positive x-Richtung) und umgekehrt. Danach berechnen wir noch die "neue" y-Richtung ‚Äî hier √§ndert sich aber nichts, also √ºbernehmen wir auf Zeile `36` einfach die aktuelle y-Richtung. Diese beiden neu berechneten Richtungswerte weisen wir dann wieder der Instanzvariable `self.direction` zu und haben damit eine neue Flugrichtung f√ºr den Ball definiert. Die Methode `flip_y_direction()` funktioniert genau gleich, nur mit den beiden Achsen vertauscht.

```python showLineNumbers {34-37,39-42,41-42,44-45,47-48,63-64,66-67}
import pygame

# Spiel initialisieren
pygame.init()

# Einstellungen f√ºr das Fenster
window_width = 800
window_height = 600

# Hintergrundfarbe festlegen
background_color = pygame.color.Color("black")

# Pygame-Fenster vorbereiten

screen = pygame.display.set_mode([window_width, window_height])

# Zugriff auf die Uhr -> damit k√∂nnen wir die Geschwindigkeit des Spiels anpassen
clock = pygame.time.Clock()

class Ball(pygame.sprite.Sprite):
    def __init__(self, x, y):
        super().__init__()
        ball_image = pygame.image.load("sprites/58-Breakout-Tiles.png")
        self.image = pygame.transform.scale(ball_image, (50, 50))
        self.rect = self.image.get_rect()
        self.rect.center = (x, y)
        self.direction = (-1, -2)

    def move(self):
        new_x = self.rect.center[0] + self.direction[0]
        new_y = self.rect.center[1] + self.direction[1]
        self.rect.center = (new_x, new_y)

    def flip_x_direction(self):
        new_x_direction = self.direction[0] * -1
        new_y_direction = self.direction[1]
        self.direction = (new_x_direction, new_y_direction)

    def flip_y_direction(self):
        new_x_direction = self.direction[0]
        new_y_direction = self.direction[1] * -1
        self.direction = (new_x_direction, new_y_direction)

    def hits_top_of_window(self):
        return self.rect.top <= 0

    def hits_side_of_window(self):
        return self.rect.left <= 0 or self.rect.right >= window_width

ball = Ball(500, 300)
balls = pygame.sprite.Group()
balls.add(ball)

run = True
while run:
    clock.tick(100)
    screen.fill(background_color)

    ball.move()
    balls.update()
    balls.draw(screen)

    if ball.hits_top_of_window():
        ball.flip_y_direction()

    if ball.hits_side_of_window():
        ball.flip_x_direction()

    pygame.display.flip()

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            run = False
```

F√ºr den unteren Rand haben wir aber bewusst keine solche Kollision definiert, denn unten fehlt uns etwas ganz anderes: ein beweglicher "Schl√§ger"; das _Paddle_. Das werden wir in den n√§chsten paar Schritten hinzuf√ºgen und ausbauen.
