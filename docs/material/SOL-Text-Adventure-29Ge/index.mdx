# SOL-Projekt: Text Adventure
- Geschichte überlegen (über Verzweigungen sprechen, eine Anzahl Verzweigungen als Richtwert vorgeben, Story Beats / Szenen statt Verwzeigungen?)
- Advanced Python bis und mit Listen und Tupel durcharbeiten
- TA ausprogrammieren
  - Code auf OneDrive, Ordner mit der Gruppe geteilt
  - Achtung: Kollaboration wird schwierig
  - Pair Programming
  - Funktionen verwenden: 1 Szene / Story Beat = 1 Funktion
  - Fehlerbehandlung: try / except, Rekursion, Gross- und Kleinschreibung ignorieren
- Anforderungen an die Handlung, das Gameplay -> eigene Page machen, mit Game Design Theory (habe ich teils schon, kann ich noch ausweiten)
  - Anzahl Szenen / Story Beats
  - Mitdenkende Spieler sollen die richtige Entscheidung treffen können
  - Spieler nicht frustrieren: Nicht 1 falsche Entscheidung, die zum Game Over führt. Stattdessen mehrere Chance geben, oder Sackgassen einbauen, aus denen man zurück kommt. Alternativ: Falscher Weg führt auch zum Ziel, ist aber mühsamer.
